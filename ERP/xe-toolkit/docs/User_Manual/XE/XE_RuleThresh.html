Introduction

The properties file XE_RuleThresh.properties contains the threshold values for the static rules contained within The XE Toolkit and used by several of the applications. Although the properties file itself contains many comments about the structure of the properties and the meaning of the values, this page will serve as an additional source of information regarding this file.

Features

Most of the properties in the file are commented out since the value they show are already used by the rule engine as a default value. For all values in the file, the following structure of the property name is used. The name of the property is suffixed with a period followed by the platform name. This allows different threshold values depending on the platform. A threshold value with no platform suffix is meaningless. For any given platform, the threshold value must contain the platform name as a suffix in order for the rule engine to find the value. Currently, in order to effect granularity at the host-name level, XE must be installed on a local disk per host so that the properties file is not shared. There is no provision for THRESHOLD.Platform.hostname nor is there any provision for a common operating system running on different machine architectures. The currently understood list of platform names are Linux, Solaris, MacOS, Windows and FreeBSD. The names are case-sensitive.
