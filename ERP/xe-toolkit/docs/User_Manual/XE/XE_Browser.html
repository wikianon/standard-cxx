Browser

Given the lack of a mechanism in Java to invoke external applications as if they were a class method, the old way of executing an application as a separate process must be used in order to leverage the capabilities of a browser within a Java application. It is further necessitated by the state of the HTML editor class in Java which is slow, incomplete, and displays unattractive output (to be charitable). Given this unfortunate summary, a properties file was needed to define the browsers on each supported platform and how to invoke it as a separate process so that HTML could be viewed quickly and attractively.

Features

There are three base properties in the TOP/etc/XE_Browser.properties file. They are BrowserNames, Path, and Invoke. The BrowserNames property is suffixed with the name of the platform, currently, Linux, Solaris, MacOS, FreeBSD or Windows. This property is a comma separated list of browser names that will be used to construct the full name of the Path and Invoke properties. The Path property is suffixed first with the name of the browser, identified in the BrowserNames property, and then by the platform name. This property is the fully qualified path name of the browser executable complete with its suffix, for instance, ".exe", on Windows. The Invoke property is also suffixed by both the browser name and the platform and it is a fully qualified path name that can be used by the native process invocation code to start the process. The XE class that uses this properties file to invoke the browser uses the Path property to find the binary in question. If it does not exist, it moves on to the next browser and its Path property. If no browser is found, an exception is thrown. The following tables enumerate all of the values currently in place in the XE_Browser.properties file.
