Usage

Top [-h remote-host{,remote-host}] [-p profile-name]

Introduction

Top, the ubiquitous process lister has once again been redeveloped into a new, more usable tool for tactical performance analysis and diagnosis. The XE version of Top is graphical and provides a network aware interface for monitoring the processes on multiple computers at once. In a single interface with a consistent look, multiple hosts running a variety of operating systems can be monitored using a static set of performance rules that determine hot spots and provide instant feedback in the process display by re-sorting the process list by the display field of most interest given the existing performance anomaly. In addition, a detailed rule panel shows the results of the rule firing and another tab displays the hardware configuration.

Features

Multi-level Tabbed Display

The top level display of Top segregates the monitored hosts into a tabbed pane. There is a separate tab pane for each monitored host and the tab area displays the name of the monitored host and a colored box indicating the condition of the host according to the static rules integrated into the application. Within each host tab pane is a toolbar and another tabbed pane that provides four areas of information about the host.

The toolbar has seven options.

  1. Interval... This will bring up an interval dialog to change the frequency of updates to the Processes display.

  2. Visible... This will change the number of processes placed into the viewport for display. The viewport is limited to the size of the screen, so a scrollbar may appear so that the other processes can be viewed.

  3. Select Fields... This will pop up a dialog with all of the possible display fields and a check-box indicating whether that field is displayed. Once the desired fields are selected and the close button is pressed, the display will be updated with the selected fields being shown.

  4. Watch Command... A dialog will pop up and allow the user to enter the name of a process to track. Once the Apply button is pressed, any process with that name will be highlighted with a light colored background and tracked as long as it exists and is on the screen. Clicking apply again with the same name will un-highlight the processes and they will no longer be tracked.

  5. Auto Sort The static rules that run continually in Top will update the process display automatically when a performance condition worse than green is encountered. The processes will be sorted by the field of the most relevance given the performance condition. For instance, if a high I/O wait time is detected, the processes will be sorted by the IO/sec field if that field is supported on the platform being monitored. If it is not supported, then the processes are sorted by the SYS% field if supported and so on. Selecting this check will cause the process display to be updated automatically based on the results of the rule firing.

  6. Sys Procs Ordinarily, the system processes are not listed in the process display. A process is considered a system process if its SIZE field is zero. Selecting this check box will cause system processes to be displayed.

  7. Close will close that host tab. If it is the last tab, it will close the window. If that is the last window, the application will exit.

There are four tabs within each host tab pane.

  1. The Processes tab shows the process display with headings for each field. Any of the heading fields can be clicked to cause sorting by that field. Clicking again will reverse the order of the sort. A red arrow appears next to the name of the field pointing down for a descending (high to low) sort or up for an ascending (low to high) sort. Any process in the display can be clicked on to mark it for tracking. The line will be redrawn with a light colored background color and will stay that way until unselected by clicking on it again, or until the process exits.

  2. The XE Gadgets tab shows the gadgets available for display. Currently, they are a CPU Pie Chart, a Memory Pie Chart and a Page File Pie Chart. Single-clicking on any of these will bring the gadget up in the lower pane and will run according to the interval set for that gadget. To change the interval for a running gadget, place the pointer on the running gadget and click the right button. For Mac users, use control-click. A popup menu will display "Interval..." and when selected, the interval dialog will come up and the interval for that gadget can be selected.

  3. The System Info tab shows a tree structure with the configuration of the monitored host split up into functional areas. Any node with a plus sign (+) can be expanded to view more information. This display also updates on an interval, and like the gadgets, a right-click or control-click will bring up an interval dialog.

  4. The Rules tab shows all of the static rules being used on that monitored host. All of the information collected or generated by that rule is displayed in its own box. The color of the border around the rule indicates the state of the rule at the last rule firing.  As with the other tabs, the interval can be changed with a right-click or control click to display the interval dialog.

Multi-Host Monitoring

Top can monitor many hosts at once. These hosts can all appear in one window, each host in its own tab, or they can be split up among different windows. Each host can have its own window by selecting the "New Top Instance..." option from the Top menu or hosts can be split up according to their platform, function, membership in a cluster or whatever criteria the user desires. If split up according to some criteria, the title in the title-bar can be changed to signify the purpose of the segregation by using the Set Title... option in the Top menu. Examples of titles are "Top: Unix Hosts" or "Top: Database Servers". Each host display can be changed to display different fields, sort ascending or descending on different fields, have different gadgets running, have different nodes open on the System Tree tab, many different options, all unique to the individual host. To add a new host tab to the existing window, select the Monitor... option in the Top menu. Whether using the Monitor… option or the “New top Instance…” option, the specified host name can be one name or a comma-separated list of names. The name can either be the name of a host or it can be the name of a pre-defined cluster (Clusters are defined later.)

Multi-Process Selection

Instead of marking just one process to watch in the display, like Task Manager in Windows, several processes can be marked for tracking or the name of the process can be entered and all processes with that name will be selected in the display and tracked. To enter a process name, use the “Watch Command…” button on the toolbar.

Cluster Definition

Regardless of whether a group of hosts are in a cluster (as defined by a cluster-aware operating system), they can be defined as a logical cluster in Top. Once the definition has been entered, the cluster name can be used as a host name to monitor. A new Top window can be opened with the cluster name and that window will contain one host tab for each host in the cluster. The cluster name can also be used on the command line in place of a remote-host.

Application Profiles

When Top has been modified to display all the information about all the hosts the way the user wants it displayed, the context of the application can be saved as a profile. This will cause Top to take a "snapshot" of itself and save it to the user's home directory in a file called ".xetopconfig". This file is a Java serialization data file which contains properties about all of the components of the application. A profile can be loaded from the Tools menu or the profile can be restarted from the command line by using the -p flag to Top. This functionality allows for a complex set of modifications to be made and then saved for reloading later without the need to revisit all of the actions.

Rule-Based Monitoring

In addition to providing feedback about the firing of static rules within the application, Top also provides a dialog with information about the most recent rule condition worse than the green state (with the exception of when Top starts and finds the system in a green state). All of the information collected by the rule when it was in that state is displayed in the dialog. It’s important to keep in mind that this dialog can falsely accuse an application of being the transgressor in a rule firing. This is because the real process that caused the problem state started and ended within the interval of the rule firing but caused enough havoc for the metrics to show a problem. When the process listing is sorted by the field of interest, the top process in the sort will be one that is still running and may very well have the highest value for that field, but not be the process that caused the problem since it had already exited.

The rules contained within Top use threshold values to set boundaries for the conditions that the rules are searching for. These threshold values are contained within the properties file XE_RuleThresh.properties.

In addition to the Interval... popup, the Disks and Network Interfaces rules have a detail dialog that can be brought up by left-clicking on the respective rule box on the Rules tab pane.
