Introduction

You have installed The XE Toolkit on your system. This package contains a server process that collects performance data and delivers it to the client applications that connect to it. The clients and server are all written in Java. The communication mechanism is RMI, the Remote Method Invocation. The connections can be made secure by modifying the file TOP/etc/CaptiveCollector.properties. Instructions on how to use this file can be found in the section on configuration.

The server process is called CaptiveCollector. The Unix startup script for this process is in the init.d directory. It is written like an rc script, that is, you can start the server by running "CaptiveCollector start". If your system supports the rc.d or init.d style startup scripts, this script can be placed in the directory representing the init state that you would like the collector started in. It should be installed as a startup script and optionally as a kill script in another init-state directory. The Windows startup program is in the system directory. It does not take any arguments. Running it will bring up a GUI called CaptiveControl that will allow you to start and stop CaptiveCollector. The file INSTALL.txt has more instructions on this topic. Before starting CaptiveCollector on any operating system, you must verify that the path to the Java 1.5 or newer programs are in the PATH variable or you can modify the Unix script to include the path at the start of the script.

The fastest way to get going is to follow the following commands. It is best to start the collector as root on Unix and Administrator on Windows. Some Unix systems will require root access. Others will provide more data if run as root and others do not need it at all. Captive Metrics recommends that the collector be run with executive privilege.

On Unix

# cd init.d

# CaptiveCollector start

# cd ../bin

# SysInfo

or on Windows

C:\> cd \Program Files\Captive Metrics\XE Toolkit\system

C:\P~C~X~S:-> CaptiveCollector

The CaptiveControl GUI will start. Press the Start button to start CaptiveCollector. Note that this may take 15 to 30 seconds, so be patient.

C:\P~C~X~S:-> cd ..\Windows

C:\P~C~X~B:-> SysInfo
